Build a GROMACS Molecular Dynamics Simulation Agent that runs as an
interactive chatbot in a Linux terminal, powered by a lightweight RAG
architecture optimized for Gemini's free API tier.

## Overview
A Python CLI chatbot where a user types natural language descriptions of
molecular dynamics simulations. Before each Gemini Flash API call, relevant
GROMACS knowledge is retrieved from a local vector database and injected
into the prompt. The knowledge base is scraped directly from the official
GROMACS documentation during ingest.py so the agent always has accurate,
real commands — never hallucinated ones.

- If the key fails, allow retry — MAX 3 attempts then exit cleanly:

## Documentation Scraping Pipeline (inside ingest.py)

### Overview
ingest.py has two phases:
  Phase 1 — Scrape: fetch GROMACS docs → save as clean markdown to docs/
  Phase 2 — Embed: chunk docs/ → embed → store in ChromaDB

### Scraping Implementation (scraper.py)
Use requests + BeautifulSoup4 + html2text. No Scrapy, no Selenium.

For each target URL:
1. GET the page with requests (timeout=15, headers with a User-Agent)
2. Parse HTML with BeautifulSoup, extract only the main content tag:
   look for <div class="document">, <article>, or <main> in that order
3. Remove all navigation, sidebar, footer, header, and breadcrumb tags:
   remove: nav, .headerlink, .toctree-wrapper, footer, .related,
           .sphinxsidebar, script, style tags
4. Convert the cleaned HTML to markdown using html2text:
   - ignore_links=False (keep command references)
   - body_width=0 (no line wrapping)
   - protect_links=True
5. Post-process the markdown:
   - Strip excessive blank lines (max 2 consecutive)
   - Remove "Table of Contents" sections
   - Ensure code blocks use triple backticks
6. Save to docs/{name}.md (e.g. docs/mdp_options.md)
7. Print live status for each page:
     Scraping mdp_options...    ✓  (48KB → docs/mdp_options.md)
     Scraping force_fields...   ✓  (12KB → docs/force_fields.md)

### Scrape Cache (scrape_cache/)
- Before fetching any URL, check scrape_cache/{name}.html for a cached
  copy alongside its timestamp
- Only re-fetch if the cached file is older than 7 days
- Save the raw HTML to scrape_cache/{name}.html after each successful fetch
- This means ingest.py can be re-run without hitting the internet again
  unless the cache is stale
- Show cache status per page:
     Scraping mdp_options...    ⚡ Using cache (2 days old)
     Scraping force_fields...   ↓  Fetching (cache expired)

### Scraping Error Handling
- If a page returns a non-200 status: log a yellow warning, skip that
  page, continue with the rest — never abort the full scrape for one
  failed URL
- If a page times out: retry once after 3s; if it fails again, skip
- If the main content tag is not found: fall back to scraping the full
  <body> minus nav/footer tags
- If ALL pages fail to scrape (no internet): check if docs/ already
  exists from a previous run and use it as-is with a warning:
      "⚠ Could not scrape docs — using existing docs/ from last run."
  If docs/ is also missing, exit with:
      "✗ No documentation available. Check your internet connection
       and re-run ingest.py."
- Never leave a partially written .md file — write to a temp file first,
  then rename atomically

### Phase 2: Embedding (same as before)
After all docs are scraped and saved to docs/:
- Load docs/*.md
- Split into 300-token chunks, 30-token overlap (tiktoken)
- Check embeddings_cache.pkl — skip chunks from files whose mtime
  has not changed since last embed
- Embed new/changed chunks with text-embedding-004
- Store in ChromaDB (chroma_db/)
- Show live progress bar with step status
- Print summary: pages scraped, chunks embedded, total time

## Error Handling — No Infinite Loops
Every error type has a fixed retry cap and returns control to the user.

- 429 Rate limit: retry 3x with backoff (5s, 10s, 20s) then return to prompt
- 401 Invalid key: print error, return to prompt, no retry
- 500/503: retry once after 3s, then return to prompt
- Timeout (all requests timeout=15s): print error, return to prompt
- ChromaDB missing: print "Run: python ingest.py", exit cleanly
- ChromaDB query failure: skip retrieval, warn, continue with empty context
- Embedding failure: skip retrieval for this turn, warn, continue
- Empty input: silently re-prompt
- Unknown /command: print help hint, return to prompt
- KeyboardInterrupt: catch, print "Use /exit to quit.", re-prompt
- Ctrl+D / EOF: catch, print "Goodbye!", exit cleanly
- Global safety net: top-level try/except around the chat loop,
  print one-line error, re-enter prompt
- MAX_CONSECUTIVE_ERRORS = 5 circuit breaker: if 5 back-to-back errors
  occur, print instructions and exit cleanly. Reset on any success.

## Lightweight & Speed Optimizations
- gemini-2.0-flash-lite for generation
- text-embedding-004 for embeddings
- K=3 retrieved chunks, 300-token chunks, 4-turn history
- Hard 3000-token prompt budget
- Stream all generation responses
- ChromaDB in read-only mode at chat startup
- Scrape cache prevents re-fetching on repeated ingest runs
- Embedding cache prevents re-embedding unchanged docs
- No LangChain, LlamaIndex, transformers, or torch

## PDB File Handling
- No automatic downloads — user places .pdb files in input_structures/
- Startup scan lists detected files
- All commands reference input_structures/filename.pdb
- /structures command rescans mid-session

## RAG Architecture

### Retrieval (retriever.py)
- top K=3, cosine similarity threshold 0.75
- On failure: return empty list, log warning, never raise to caller

### Prompt (prompt_builder.py)
- Hard 3000-token budget
- Trim oldest history first if over budget, then lowest-scored chunk
- Format: SYSTEM → CONTEXT → HISTORY → QUESTION

## Terminal UI
- Stream Gemini responses token by token
- Compact spinner only during embedding
- One-line source citation after each response:
    📄 Sources: mdp_options.md · gmx_commands.md
- Commands: /help /clear /save /sources /reindex /structures /exit /key
- Enter twice to submit; single Enter for newline
- ASCII art banner on startup

## GROMACS Agent Behavior (system prompt)
- Answer only from retrieved context — no hallucinated parameters
- Be concise — no lengthy preambles
- Generate complete .mdp files and bash pipelines
- Reference input_structures/ in all commands
- Cite source documents inline
- GROMACS 2023/2024 syntax
- Never download structure files
